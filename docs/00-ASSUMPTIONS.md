# 가정 문서 (Assumptions)

> 작업 중 막힌 지점에서 세운 가정들. **사용자 확인이 필요한 항목은 docs/07-DECISIONS-FOR-USER.md에 요약**되어 있습니다.
> 가정이 틀렸다면 해당 모듈만 교체하면 되도록 코드를 구성했습니다.

## A1. 기존 코드 재사용 불가 → 공식 스펙 기반 신규 작성

- **상황**: CLAUDE.md에 "기존 KIS API 연동 코드, KR-FinBert-SC 모델 재사용"이 명시되어 있으나,
  이 작업 환경(이 디렉토리/공유 메모리)에서 기존 KATT 코드에 접근할 수 없음.
- **가정**: KIS Developers 공식 OpenAPI 스펙(oauth2 토큰, 국내/해외 시세 조회)에 맞춰
  새로 구현하되, `backend/app/data/kis.py` 한 파일에 격리. 기존 코드가 있다면 이 파일만
  교체하면 됨. KR-FinBert-SC는 HuggingFace `snunlp/KR-FinBert-SC` 기준으로 통합
  (기존 KATT에서 쓰던 것과 동일 모델로 가정).

## A2. 개발 머신에 Xcode 없음 → 시뮬레이터 검증 대체

- **상황**: 이 머신에는 CommandLineTools만 설치되어 있어 `xcodebuild`/`simctl` 사용 불가.
- **가정**: iOS 코드는 (1) 전체 SwiftUI 소스 + (2) XcodeGen `project.yml` + (3) 실기기/시뮬레이터
  설치 가이드(docs/06)로 전달. Swift 문법 검증은 `swiftc -parse`로 수행.
  사용자 머신에 Xcode가 설치되어 있다고 가정 (사이드로딩 계획이 이미 있으므로).

## A3. 푸시 알림: 무료 Apple ID 사이드로딩은 APNs 불가 → 로컬 알림 + 백그라운드 갱신

- **상황**: 무료 Apple ID 프로비저닝은 Push Notifications entitlement를 지원하지 않고,
  앱 서명이 7일마다 만료됨. 유료 개발자 계정($99/년)이면 APNs 가능 + 1년 서명.
- **가정(기본 구현)**: 백엔드는 `notification_events` 테이블에 알림 이벤트만 적재하고,
  iOS 앱이 **BGAppRefreshTask(백그라운드 갱신) + 로컬 알림**으로 전달. 무료 계정에서도 동작.
  - 한계: iOS가 백그라운드 갱신 시점을 보장하지 않음 (보통 하루 수회). 일간 신호 용도로는 충분.
  - 유료 계정 보유 시 APNs로 업그레이드 가능하도록 백엔드에 이벤트 구조를 이미 분리해 둠.
- **사용자 판단 필요**: 유료 개발자 계정 보유 여부 (docs/07 참고).

## A4. 섹터 프록시: 개별 종목이 아닌 대표 ETF로 섹터를 측정

- **가정**: 신호는 "섹터 단위"이므로 섹터별 대표 ETF의 가격/거래량을 지표로 사용.
  종목 선택은 사용자 몫. ETF 매핑은 `backend/app/engine/sectors.py`에서 상수로 관리 (교체 쉬움).

| 섹터 | 한국 (KRX) | 미국 (NYSE/NASDAQ) |
|---|---|---|
| 반도체 | KODEX 반도체 (091160) | SMH (VanEck Semiconductor) |
| 로봇 | KODEX K-로봇액티브 (445290) | BOTZ (Global X Robotics & AI) |
| 전력 | KODEX AI전력핵심설비 (487240) | GRID (First Trust Smart Grid Infra) |
| 헬스케어 | TIGER 헬스케어 (143860) | XLV (Health Care) |
| 금 | ACE KRX금현물 (411060) | GLD (SPDR Gold) |
| 국채 | KOSEF 국고채10년 (148070) | TLT (20Y+) / IEF (7-10Y) |

- **"전력" 확정 (2026-07-02 사용자 확인)**: 유틸리티(한전)가 아니라 **전력기기/인프라**
  (변압기·중전기 등 AI데이터센터發 전력설비 수혜주). KR은 KODEX AI전력핵심설비(487240,
  2024-07 상장) — 효성중공업/HD현대일렉트릭/LS ELECTRIC 비중 약 69%. US는 XLU(유틸리티)

  2009년 상장이라 장기 백테스트 가능)로 교체. 재보정 결과는 docs/04 §7.
  → **해결됨** (기존 docs/07 항목 1 클로즈).

## A5. 백테스트는 미국 ETF + FRED 데이터로 프레임워크를 검증

- **상황**: 백테스트용 장기 일봉을 키 없이 안정적으로 받을 수 있는 소스가 stooq(미국 ETF)와
  FRED CSV(금리/VIX/스프레드)뿐. 한국 ETF 장기 일봉은 KIS 키 필요.
- **가정**: 스코어링 프레임워크(국면 분류, 가중치, 임계값)는 미국 데이터 10년으로 검증하고,
  같은 공식을 한국 섹터에 적용(지표 소스만 교체). 한국 측 검증은 KIS 키 투입 후
  `backend/backtest/`를 재실행하면 됨.

## A6. 미국 뉴스 감성: Alpha Vantage NEWS_SENTIMENT를 1차 소스로 채택

- **근거**: (1) 무료 티어 25 req/day — 일 1회 배치에 충분, (2) 기사별 감성 점수+티커 관련성
  포함이라 별도 모델 호스팅 불필요, (3) LLM 호출 0회.
- **대안(코드에 인터페이스로 분리)**: Finnhub news-sentiment, 또는 ProsusAI/FinBERT를
  Railway 워커에서 배치 실행 (메모리 ~1.5GB 필요, 비용 증가).
- **사용자 액션 필요**: Alpha Vantage 무료 API 키 발급 (이메일만 있으면 1분).

## A7. 거시 국면의 "글로벌 축"은 미국 지표 기준

- **가정**: 위험선호/회피 축은 글로벌 공통(미국 S&P500, VIX, HY스프레드 기준)으로 판정.
  한국 시장은 글로벌 국면을 따르되, **시장별 하위상태(로컬 추세)** 로 KOSPI 자체 추세를
  반영. 근거: 한국 주식 섹터의 체계적 위험은 글로벌(미국) 리스크 사이클에 지배되고,
  로컬 요인은 추세/수급에 이미 반영됨. 금리 축은 시장별로 각자 판정 (한국: ECOS 국고채,
  미국: FRED).

## A8. 신호 전환 빈도 억제: 2일 연속 확인 + 히스테리시스 + 최소 유지 5거래일

- **가정**: 개인 수동 매매 특성상 잦은 신호 전환은 해로움. 전환 조건을 만족해도
  2거래일 연속 유지될 때만 신호 변경, 전환 후 5거래일 내 재전환 금지(단, 현금보유 전환은
  리스크 방어라 3거래일로 완화). 백테스트에서 이 값들을 검증/튜닝함.

## A9. Supabase/Railway 프로젝트 생성은 사용자 액션

- **상황**: 새 Supabase/Railway 프로젝트 생성에는 사용자 계정 로그인이 필요.
- **가정**: 마이그레이션 SQL, railway.toml, 환경변수 목록, 단계별 가이드(docs/05)를 제공하고
  실제 프로젝트 생성/키 발급은 사용자가 수행. 로컬에서는 Supabase 없이도 SQLite 폴백으로
  전체 파이프라인이 돌도록 구현 (개발/검증용).

## A10. 구현 세부 가정은 모듈별 노트 참고

- 백엔드 구현 세부 가정 10건: `backend/AGENT_NOTES.md`
  (핵심: 신호 변경/알림의 기준은 스탠스(LONG↔CASH) 전환 — hold↔keep 진동은 알림 아님.
  백필은 z-score 워밍업 252거래일 이후부터 저장. ECOS 무키 시 KR 금리를 미국 금리로 대체하고 degraded 표기)
- iOS 구현 세부 가정 15건: `ios/AGENT_NOTES.md`
  (핵심: news sentiment는 수치(±0.15 중립 밴드), 다이제스트는 다음 도래 시각에 1건으로 묶음)

## A11. 시간대/스케줄

- **가정**: 배치는 하루 2회 — ① 한국장 마감 후 15:50 KST (한국 섹터 갱신 + 국면 재계산),
  ② 미국장 마감 후 06:10 KST (미국 섹터 갱신 + 국면 재계산 + 아침 다이제스트 생성).
  뉴스 감성은 ②에서 함께 수집. 장중 실시간 갱신은 하지 않음 (일봉 기반 신호이므로).
