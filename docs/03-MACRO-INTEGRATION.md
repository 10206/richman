## 2. 금리 변화의 점수 반영 — "수준이 아니라 방향과 속도"

금리 **수준**은 국면에 따라 의미가 달라 직접 쓰지 않고, **변화율(방향)** 과
**변화속도(z-score)** 만 사용:

1. **방향**: 63일(3개월) 변화 — L축 국면 판별 (docs/01)
2. **속도**: 변화량의 2년 z-score — 같은 25bp 상승도 저변동 시기엔 큰 신호, 2022 같은
   시기엔 평범한 신호가 되도록 정규화
3. **섹터 전달 경로**:
   - 국채: `rate_score` 직접 반영 (하락 = 호재)
   - 금: 명목금리 아닌 **실질금리**(DFII10) 변화 사용
   - 성장주(반도체/로봇): ±50bp/3개월 초과 급변 시에만 `rate_sensitivity` 페널티/보너스
     (평상시 금리 미세변동에 주식 점수가 흔들리지 않도록 **불감대(deadband)** 설계)
   - 방어주(전력/헬스케어): 같은 로직을 절반 강도로

## 3. 뉴스 감성 — 노이즈 vs 유의미 신호 구분

원칙: **단발성 기사에 반응하지 않고, "폭(breadth) × 지속(persistence) × 강도(magnitude)"
3중 게이트를 모두 통과한 감성만 점수에 반영.**

```
일간 섹터 감성 s_t = Σ(기사 감성 × 관련성 가중) / Σ(관련성 가중)    ∈ [-1, +1]

fast_t = EWMA(s_t, halflife=3일)      # 최근 흐름
slow_t = EWMA(s_t, halflife=21일)     # 베이스라인
z_t    = (fast_t − slow_t) / rolling_std(s, 63일)
```

| 게이트 | 조건 | 걸러내는 것 |
|---|---|---|
| ① 폭 | 최근 3일 관련 기사 수 ≥ 5 | 기사 1~2건짜리 단발 이슈 |
| ② 강도 | \|z_t\| ≥ 1.0 | 베이스라인 대비 평범한 등락 |
| ③ 지속 | z_t 부호가 2일 연속 동일 | 하루짜리 헤드라인 스파이크 |

- 3중 게이트 통과 시: `news = squash(z_t)` (M 컴포넌트의 15% — 상한 자체가 작음)
- 미통과 시: `news = 0.5` (중립) — **점수에 영향 0**
- 지정학 리스크(금 전용): 제목 키워드 필터(전쟁/제재/공습 등) 매칭 기사가 3일 5건 이상이면
  금의 risk_aversion_score에 +0.1 보정

## 4. LLM 사용 구분 (비용 원칙)

| 작업 | 처리 방식 | LLM |
|---|---|---|
| 국면 판별, 점수/신호 계산 | 순수 수식, 일 2회 배치 | **없음** |
| 한국 뉴스 감성 점수 | KR-FinBert-SC 배치 (로컬 모델) | 없음 (BERT) |
| 미국 뉴스 감성 점수 | Alpha Vantage API | 없음 |
| 알림 본문 한 줄 이유 | 템플릿 + 컴포넌트 변화량 (규칙 기반) | 없음 |
| 섹터 상세 화면 "관련 뉴스 요약" | 일 1회 배치, Claude Haiku, 프롬프트 캐싱 | **Haiku (일 6섹터×2시장 = 12콜/일)** |
| 거시 이벤트 해석 (FOMC/금통위 요약) | 이벤트 발생일에만 배치, Haiku | Haiku (월 2~4콜) |

- 인앱 실시간 LLM 호출: **0** (앱은 백엔드 DB에 저장된 배치 결과만 읽음)
- 예상 LLM 비용: Haiku 기준 월 $1 미만
- 뉴스 요약 실패/미설정 시에도 신호 파이프라인은 영향 없음 (요약은 표시용 부가 기능)
