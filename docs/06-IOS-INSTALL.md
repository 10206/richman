# iOS 앱 빌드 & 실기기 설치 가이드

## 준비물

- Xcode 15 이상 (App Store에서 설치)
- XcodeGen: `brew install xcodegen`
- 아이폰 + USB 케이블 (또는 같은 Wi-Fi)

## 1. 프로젝트 생성 & 시뮬레이터 실행

```bash
cd ios
xcodegen generate        # RichSignal.xcodeproj 생성
open RichSignal.xcodeproj
```

Xcode에서 시뮬레이터(iPhone 15 등) 선택 → **⌘R**.
기본값이 **모의 데이터 모드**라 백엔드 없이도 전체 화면(대시보드/상세/설정)이 바로 동작한다.
백엔드 연결: 설정 탭 → 모의 데이터 끄기 → 서버 URL/API 키 입력.

## 2. 실기기 설치 (사이드로딩)

1. Xcode → 프로젝트 선택 → **Signing & Capabilities**
   - Team: 본인 Apple ID 추가 (Xcode → Settings → Accounts → +)
   - Bundle Identifier가 겹치면 `com.<본인아이디>.richsignal` 로 변경
2. 아이폰을 케이블로 연결 → 상단 기기 선택 → **⌘R**
3. 아이폰에서: 설정 → 일반 → VPN 및 기기 관리 → 개발자 앱 → 신뢰

### 무료 Apple ID 사용 시 제약 (가정 A3)

| 항목 | 무료 계정 | 유료 계정 ($99/년) |
|---|---|---|
| 서명 유효기간 | **7일** (매주 재설치 필요, 케이블 연결 후 ⌘R 한 번이면 됨) | 1년 |
| 원격 푸시(APNs) | 불가 → **로컬 알림 + 백그라운드 갱신**으로 동작 (기본 구현) | 가능 |
| 기기 수 | 3대 | 100대 |

앱은 무료 계정 기준으로 설계됨: 백그라운드 갱신(iOS가 하루 수회 깨워줌) 시
서버의 알림 이벤트를 가져와 로컬 알림으로 표시. "현금보유 전환"은 가져온 즉시 배너,
그 외는 설정한 다이제스트 시각(기본 07:30)에 묶어서 1건으로 표시.

> 백그라운드 갱신은 iOS 재량이라 실시간성이 필요하면: ① 아침/저녁에 앱을 한 번 열기
> (포그라운드 진입 시 항상 동기화), ② 또는 유료 계정 + APNs 업그레이드 (docs/07 참고).

## 3. 알림 권한

첫 실행 시 알림 권한 요청에 **허용**.
아이폰 설정 → RichSignal → **백그라운드 앱 새로 고침 켜기** (알림 수신에 필요).

## 문제 해결

- `xcodegen: command not found` → `brew install xcodegen`
- 기기에서 "신뢰할 수 없는 개발자" → 위 2-3단계 신뢰 설정
- 7일 후 앱 실행 안 됨 (무료 계정) → 케이블 연결 후 Xcode ⌘R로 재설치 (데이터 유지됨)
- 서버 연결 실패 → 설정 화면에서 URL에 https:// 포함했는지, API 키 일치하는지 확인
