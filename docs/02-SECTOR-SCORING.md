# 섹터별 스코어링 로직

## 1. 공통 구조

모든 섹터 점수는 3개 컴포넌트의 가중합, 각 컴포넌트는 [0, 1], 총점은 [0, 100]:

```
S = 100 × (w_T·T + w_V·V + w_M·M)     (w_T + w_V + w_M = 1)
```

- **T (추세)** — 섹터 프록시 ETF의 가격 추세
- **V (거래량/수급)** — 방향성 있는 거래량 (거래량 급증 자체가 아니라 "어느 방향으로의 급증인지")
- **M (거시)** — 국면 bias + 섹터 특화 거시 지표 + 뉴스 감성

z-score는 **스케일 정규화 전용** (`z = x / rolling_std(x)`, ±4 클리핑, 2년=504거래일 윈도우) —
평균을 빼지 않는다. 입력이 이미 부호가 의미 있는 변화량이므로 평균을 빼면 지속 추세가
중립으로 왜곡되기 때문. `squash(x) = (tanh(x) + 1) / 2` 로 [0,1] 변환.

### T — 추세 컴포넌트 (공식 공통, 룩백은 섹터 유형별 fast/slow)

```
t_raw = 0.40·z(P/MA_base − 1) + 0.30·z(MA_short/MA_base − 1) + 0.30·z(ret_n)
T = squash(t_raw)
fast (반도체/로봇):        MA_base=60,  MA_short=20, ret_n=63
slow (전력/헬스케어/금/국채): MA_base=120, MA_short=60, ret_n=126
```

- `P/MA_base`: 중기 추세 대비 현재 위치 (추세 강도)
- `MA_short/MA_base`: 정배열 여부 (추세 방향 전환 감지)
- `ret_n`: 모멘텀 (횡단면 모멘텀 연구에서 검증된 룩백)
- fast/slow 구분 근거: 평균회귀 성향 섹터에 fast를 쓰면 왕복 손실 발생 (docs/04 §4)

### V — 거래량 컴포넌트 (전 섹터 공통 공식)

거래량은 방향과 결합해야 정보가 됨 (급증 자체는 바닥/천장 양쪽에서 발생):

```
uvr   = 최근 n일 중 상승일 거래량 합 / 전체 거래량 합         # up-volume ratio, 0.5 = 중립
obv_z = z(OBV의 n일 변화)                                      # 수급 방향
vol_z = z(n일 평균 거래량)                                     # 관심도 (보조)
v_raw = 2.5·(uvr − 0.5)·(1 + 0.3·clip(vol_z, 0, 2)) + 0.5·obv_z
V = squash(v_raw)                        # n: fast=21, slow=42
```

- 거래량 급증(vol_z>0)은 uvr 신호를 **증폭**하는 역할만 함 (독립 신호 아님)
- 근거: OBV/거래량 가중 모멘텀은 추세 확인 지표로서 단독 사용보다 결합 사용이 유효

## 2. 가중치 — 섹터 유형 × 국면

### 기본 가중치 (섹터 유형별)

| 섹터 유형 | w_T | w_V | w_M | 근거 |
|---|---|---|---|---|
| 주식 섹터 (반도체/로봇/전력/헬스케어) | 0.45 | 0.25 | 0.30 | 섹터 ETF 수익의 1차 설명변수는 자체 추세. 거시는 국면 전환기에만 지배적 |
| 국채 | 0.25 | 0.10 | 0.65 | 채권 가격 = 금리의 함수. 거래량 정보가치 낮음 |
| 금 | 0.30 | 0.10 | 0.60 | 실질금리·달러가 구조적 동인 (학술 합의). 추세는 보조 |

### 국면별 조정 (주식 섹터만)

| 국면 | w_T | w_V | w_M | 근거 |
|---|---|---|---|---|
| G, R (위험선호) | 0.50 | 0.25 | 0.25 | 상승장에서는 추세 추종이 유효 (모멘텀 지속성) |
| T, F (위험회피) | 0.35 | 0.20 | 0.45 | 위기 국면에서는 개별 추세보다 거시가 지배 (상관계수 1로 수렴 현상). 방어가 우선이므로 거시 비중 상향 |

국채/금은 국면과 무관하게 고정 (이미 거시 지배적).

## 3. 섹터별 M 컴포넌트

### 국채 (금리 방향이 전부)

```
M_bond = 0.45·rate_score + 0.25·curve_score + 0.30·risk_aversion_score
```

| 항목 | 계산 | 근거 |
|---|---|---|
| rate_score | squash(−z(10년물 63일 변화)) — 금리 하락 = 높음 | 듀레이션 손익의 직접 동인 |
| curve_score | squash(z(10Y−2Y 스프레드의 63일 변화)) — 스티프닝 국면에서 금리인하 사이클 감지 | 불 스티프닝 = 인하 사이클 = 채권 강세 신호 |
| risk_aversion_score | 1 − squash(R_score×1.5) — 위험회피일수록 높음 | 안전자산 도피 수요 |

### 금 (실질금리 + 달러 + 위험회피)

```
M_gold = 0.40·real_rate_score + 0.30·dollar_score + 0.30·risk_aversion_score
```

| 항목 | 계산 | 근거 |
|---|---|---|
| real_rate_score | squash(−z(10Y TIPS 실질금리 63일 변화)) (FRED DFII10) | 금 보유의 기회비용. 금 가격의 최우선 설명변수 |
| dollar_score | squash(−z(달러인덱스 63일 변화)) (FRED DTWEXBGS) | 달러 표시 자산 — 역상관 |
| risk_aversion_score | 1 − squash(R_score×1.5), 지정학 뉴스 감성으로 ±0.1 보정 | 위기 헤지 수요 (지정학 리스크는 뉴스 키워드 기반) |

### 주식 섹터 (반도체/로봇/전력/헬스케어)

```
M_equity = 0.50·M_base(국면 bias) + 0.25·local_trend + 0.15·news + 0.10·rate_sensitivity
```

| 항목 | 계산 |
|---|---|
| M_base | (국면 bias + 2) / 4 — docs/01의 매트릭스 |
| local_trend | 해당 시장 하위상태: 강세 1.0 / 중립 0.5 / 약세 0.0 |
| news | 섹터 뉴스 감성 필터 통과값 (docs/03) — 통과 못 하면 0.5 (중립) |
| rate_sensitivity | 금리 급변 페널티/보너스: 10년물 63일 변화가 +50bp 초과 시 성장섹터(반도체/로봇) 0.0, −50bp 미만 시 1.0, 그 외 0.5. 전력/헬스케어는 ±0.25 강도로 완화 적용 |

## 4. 신호 규칙 (임계값 + 히스테리시스 + 확인)

내부 스탠스는 2상태(**LONG** / **CASH**), 표시 신호는 3종:

```
점수 S ≥ enter (confirm일 연속)      → 스탠스 LONG,  신호 🟢▲ 보유
점수 S ≤ exit  (confirm일 연속)      → 스탠스 CASH,  신호 🔴■ 현금보유
exit < S < enter                     → 스탠스 유지,   신호 🟡▶ 유지
```

섹터별 확정 파라미터 (백테스트 그리드 서치, docs/04 §4):

| 섹터 | enter | exit | confirm | 설계 의도 |
|---|---|---|---|---|
| 반도체/로봇 | 55 | 35 | 2일 | 고베타 — 빠른 방어 전환 |
| 전력 | 60 | 30 | 2일 | 전력기기/인프라(GRID/487240) — 성장형 재보정 (docs/04 §7), 상시보유 성향은 유지하되 진입선 상향 |
| 헬스케어 | 50 | 30 | 3일 | 방어섹터 — 사실상 상시 보유, 극단에서만 현금 |
| 금 | 55 | 35 | 3일 | 거시 주도 — 노이즈 억제 |
| 국채 | 50 | 35 | 3일 | 2022형 금리쇼크 방어가 목적 |

- **히스테리시스**: 진입/이탈 임계값 분리(15~20점 폭)로 경계 근처 왕복 매매 차단
- **확인 기간**: 하루짜리 스파이크 무시
- **쿨다운**: 전환 후 5거래일 내 재전환 금지. 단 LONG→CASH(방어 방향)는 3거래일로 완화
  — 손실 방어는 빠르게, 재진입은 신중하게 (비대칭 설계)
- 철학: "기본은 보유, 명백히 나쁠 때만 현금". 초안 65/40은 중립 구간 현금 대기가 길어
  상승분 손실이 컸음 (docs/04 §4).

### 알림 이벤트 매핑 (CLAUDE.md 원칙)

| 이벤트 | 전달 방식 |
|---|---|
| LONG → CASH (현금보유 전환) | **즉시** |
| CASH → LONG (보유 전환) | 다이제스트 |
| 거시 국면 변경 | 다이제스트 |
| 점수 변동 (신호 불변) | 알림 없음 |

알림 본문 한 줄 이유는 **가장 크게 변한 컴포넌트**에서 자동 생성 (LLM 아님):
예) `반도체: 보유→현금보유 (T 국면 진입 + 추세 점수 71→38)`
