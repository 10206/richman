# 프로젝트: 개인용 매매신호 iOS 앱

## 목적
- 실제 자동매매가 아님. 신호만 제공하고, 매매 실행은 사용자가 수동으로 함.
- 개인 1인 사용. 앱스토어 배포 안 함. Xcode로 직접 기기에 설치(사이드로딩).

## 대상 시장 / 섹터
- 시장: 한국(KRX), 미국(NYSE/NASDAQ)
- 섹터: 반도체, 로봇, 전력, 헬스케어, 금, 국채

## 아키텍처 원칙
- 국면 분류 + 점수 계산은 순수 로직(파이썬 함수/수식)으로 처리. LLM 실시간 호출 금지.
- 뉴스/거시 이벤트 해석 등 정성적 판단에만 경량 모델(Sonnet 이하) 사용.
- 기존 인프라 재사용: KIS API, FastAPI, Railway, Supabase
  - 단, KATT(퓨전연구소) 프로덕션 인프라와는 완전히 분리된 별도 Railway/Supabase
    프로젝트로 구성할 것. 같은 계정이라도 프로젝트는 분리.

## 국면(Regime) 분석 — 하이브리드 방식
- "거시 국면"은 시장 전체에 대해 하나로 정의 (예: 금리 상승기/하락기,
  위험선호/위험회피 국면 등 — 구체적 국면 개수와 판별 지표는 설계 단계에서 확정)
- 각 섹터는 이 공통 거시 국면 안에서 "강세/약세/중립" 반응 방향을 개별 정의
  (예: 위험회피 국면 → 반도체/로봇 약세 방향, 국채/금 강세 방향,
  전력/헬스케어 중립~방어적 방향)
- 한국장과 미국장은 서로 다른 하위 상태에 있을 수 있음을 고려

## 섹터별 특성
- 국채: 금리 방향, 장단기 금리차(일드커브), 위험회피 심리가 핵심 지표
- 금: 실질금리, 달러 강세/약세(달러 인덱스), 지정학적 리스크가 핵심 지표
- 반도체/로봇/전력/헬스케어: 거래량/추세 중심 + 국면별 가중치 조정

## 거시 데이터 소스
- 한국 금리: 한국은행 ECOS API
- 미국 금리: FRED API
- 한국 뉴스 감성: 기존 KR-FinBst-SC 모델 재사용
- 미국 뉴스 감성: (설계 단계에서 경량 모델/API 확정)

## UI 설계 원칙
- 화면 정보 우선순위: ① 신호 변화 감지(액션 필요) → ② 오늘의 공통 거시 국면 요약
  → ③ 전체 섹터 현황
- 신호 색상/아이콘 고정: 🟢▲ 보유 / 🔴■ 현금보유 / 🟡▶ 유지
  (색약 사용자 고려해서 색상과 아이콘을 항상 같이 사용)
- 점수는 거래량/추세/거시요인 구성요소로 분해해서 표시 (총점만 보여주지 않음)
- 섹터 상세 화면은 드릴다운 구조 (요약 → 구성요소 → 원본 데이터)

## 알림(푸시) 설계 원칙
- 점수의 미세 변동이 아니라 "신호 자체가 바뀔 때만" 발송 (임계값 기반)
- 알림 본문에 핵심 이유 한 줄 포함
- 미국장 시간대(한국 밤~새벽)는 다이제스트로 모아서 아침에 전달
- "현금보유 전환"은 즉시 알림, 그 외는 다이제스트
